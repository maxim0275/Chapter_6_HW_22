1. Реализован механизм CRUD для модели продуктов с использованием модуля django.forms. При не разрешено присваивать продуктам названия и добавлять в описание слова, которые включены в список запрещенных слов.
Реализована валидация формы, чтобы проверять отсутствие этих слов (в любом регистре) в данных полях.

2.Добавлена кастомная валидация для поля price в форме создания и редактирования продуктов. Валидация должна проверять, что цена продукта не может быть отрицательной. Реализовано с использованием метода clean_price в форме. Если цена введена неправильно, отображается соответствующее сообщение пользователю.

3.Добавлена стилизация форм для продуктов методом __init__.
